  [http://jslint.com/](http://jslint.com/)

  [http://jsfiddle.com/](http://jsfiddle.com/)

- Babel

  [https://babeljs.io/](https://babeljs.io/)

  Babel is a JavaScript compiler. Use next generation JavaScript, today.

- Node

  [https://nodejs.org/en/](https://nodejs.org/en/)

  Node.js is a JavaScript runtime built on Chrome's V8 JavaScript engine. Node.js uses an event-driven, non-blocking I/O model that makes it lightweight and efficient.

  [https://www.npmjs.com/](https://www.npmjs.com/)

  [https://webpack.github.io/](https://webpack.github.io/)

- jspm

  [http://jspm.io/](http://jspm.io/)

- uRequire

  [http://urequire.org/](http://urequire.org/)

## Libraries
- lodash

  [https://lodash.com/](https://lodash.com/)

  A utility library delivering consistency, modularity, performance, & extras. Replaces and extends _underscore_ library (effectively superceding it).

- JQuery

  [http://jquery.com/](http://jquery.com/)

  A fast, concise, library that simplifies how to traverse HTML documents, handle events, perform animations, and AJAX.

- Modernizr

  [https://modernizr.com/](https://modernizr.com/)

  Modernizr tells you what HTML, CSS and JavaScript features the user's browser has to offer.

- es6-promise

  [https://github.com/jakearchibald/es6-promise](https://github.com/jakearchibald/es6-promise)

  This is a polyfill of the ES6 Promise, implementing a subset of *rsvp.js.

- stampit

  [https://github.com/stampit-org/stampit](https://github.com/stampit-org/stampit)

  Create objects from reusable, composable behaviors. Stampit uses three different kinds of prototypal OO to let you inherit behavior in a way that is much more powerful and flexible than classical OO.

- SystemJS

  [https://github.com/systemjs/systemjs](https://github.com/systemjs/systemjs)

  Universal dynamic module loader - loads ES6 modules, AMD, CommonJS and global scripts in the browser and NodeJS. Works with both Traceur and Babel.

- uRequire

  [http://urequire.org/](http://urequire.org/)

  With a simple build step, uRequire converts your modules to UMD (or AMD, nodejs or 'combined.js') using static code analysis, a code generation template system and build/runtime dependency path resolution along with other build & runtime goodies.
